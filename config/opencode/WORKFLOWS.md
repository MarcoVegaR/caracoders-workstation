# OpenCode Workflows

## Laravel feature

1. Read PRD/spec.
2. Inspect routes, models, migrations, policies and tests.
3. Produce a short implementation plan.
4. Implement inside the project Dev Container when possible.
5. Run formatting, tests and static checks.
6. Summarize changed files and risks.

## Workstation maintenance

1. Run `./doctor.sh`.
2. Run `./bootstrap.sh --profile <profile> --dry-run`.
3. Apply with explicit profile only.
4. Run `./verify.sh`.
